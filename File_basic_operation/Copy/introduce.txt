当终端用户想要复制一个文件，受限向客户端发送将文件复制到新路径的命 令。
  客户端向网络代理发送将文件复制到新路径的请求。
  对于对应的每个分片指 针，
  网络代理： 
   验证客户端拥有分片元数据下载的权限。 
   查找指针中存储节点的联系信息。 
   返回被请求的分片元数据、一个新的根片段 ID 以及联系信息。
  对于每个分片指针，客户端：
   用路径导出的加密密钥解密元数据。 
   将路径改为新的目的地。 
   对每个存储节点激活复制操作，从指针中用新的片段 ID 复制片段。 
   等待存储节点响应，它们已经复制了数据，并且已经删除了失败的节点。 
   用新的片段 ID 和新路径导出的新加密密钥重新加密元数据。 
   将所有修改的分片指针上传到网络代理。 
  如果存储节点实施去重存储，或者只保存一份实际的数据是允许的。最关键 的是存储节点能够通过旧的和新的片段 ID 识别数据。如果其中一个片段 ID 收到 删除操作，另一个 ID 仍然可以正常工作。只有两个片段都被删除，节点才会释 放空间
